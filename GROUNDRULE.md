# GROUNDRULE — 트레이딩 전략 전체 지침

> 이 문서는 시스템의 단일 진실 공급원(Single Source of Truth)입니다.
> 코드, 에이전트 `.md`, 오케스트레이터 모두 이 문서의 내용을 따릅니다.
> 변경 시 반드시 이 문서를 먼저 수정하고, 코드에 반영하세요.

---

## 1. 핵심 원칙

1. **손절점 도달 시 반드시 손절** — 예외 없음
2. **돈이 몰리는 섹터 안에서만 거래** — 섹터 선정이 모든 것의 출발점
3. **수익 중인 종목만 보유** — 손절을 실행하면 자동으로 달성됨

---

## 2. 종목 진입 조건 (3가지 모두 충족)

| 조건 | 기준 | 비고 |
|------|------|------|
| **1. 이동평균선 정배열** | 5일선 > 20일선 > 60일선 | 단기 > 중기 > 장기 |
| **2. 전날 종가 > 20일선** | 캔들 마감 기준 (장중 이탈 무시) | 이탈 = 20일선 아래 종가 마감 |
| **3. 60일 우상향 추세** | 고점과 저점이 지속적으로 상승 | 눌림목이 있어도 이전 저점보다 높아야 함 |

### 매수 타이밍
- 위 3가지 조건을 만족한 상태에서 **눌림목 이전의 마지막 고점(swing_high) 돌파 시** 진입
- 매수 시간대: **10:00 ~ 10:30**

### swing_high 확정 기준 _(킥오프 미팅 확정)_
> "최근 20일 내, 이후 **2거래일 이상 연속 하락**이 확인된 **가장 최근 고점봉의 고가**"
- 복수 후보 시: 가장 최근 것을 사용
- 기준 충족 봉이 없는 경우: 최근 20일 최고가로 fallback
- **fallback 적용 시**: 차트 분석가는 리스크 매니저에게 `swing_high_is_fallback=True` 플래그를 명시적으로 전달 → 리스크 매니저는 보수적 판단 적용

## 3. 손절 / 익절 기준

| 구분 | 기준 | 실행 시점 |
|------|------|-----------|
| **손절가** | 매수 시점의 20일 이동평균선 가격 **(매수 시 1회 확정, 이후 불변)** | 현재가가 손절가 이하 도달 즉시 매도 |
| **익절 대상 선정** | 현재 주가 < 20일 이동평균선 (종가 마감 기준) | 장시작 30분 전(08:30) 확인 |
| **익절 매도 실행** | 익절 대상 종목 시장가 매도 | 장시작(09:00) |

## 4. 섹터 선정 기준

- **방법**: 매일 08:00, 업종지수 **1개월 수익률 (전날 T-1 종가 기준 직전 20영업일)** 기준 최상위 섹터 1개 선정
- **데이터**: KIS API `inquire-daily-indexchartprice` (TR: `FHKUP03500100`)
- **종목 관리**: `data/sectors.json`에 섹터별 종목 저장, 직접 편집 가능

### 섹터 선정 보류 기준 _(킥오프 미팅 확정)_
| 상황 | 판단 | 팀 행동 |
|------|------|---------|

| **1위·2위 차이 1%p 미만** | `confident=True` + `warning` 발령 | 매수 가능하나 팀장이 보수적으로 판단 |
| **정상** | `confident=True` | 정상 진행 |

## 5. 리스크 관리 규칙

### 포지션 크기 계산
```
허용 손실액   = 가용 현금 × 2%
리스크 per 주 = 진입가 - 손절가
매수 수량     = 허용 손실액 ÷ 리스크 per 주
최대 수량     = 가용 현금 × 20% ÷ 진입가
최종 수량     = min(매수 수량, 최대 수량)
```

### 절대 한도
- 단일 거래 최대 손실: 가용 현금의 **2%**
- 단일 종목 최대 투자: 가용 현금의 **20%**
- 최대 보유 종목 수: **5개**
- 최소 현금 보유: **10~20%** (기회 대응 여력 확보)

### 가용 현금 기준 시점 _(킥오프 미팅 확정)_
- **10:00 포지션 계산 시** 기준 = **가장 최근 슬랙 보고에 명시된 현금 잔액**
- 09:00 익절 매도 완료 시 슬랙 메시지에 "현금 잔액: OOO원" 반드시 포함
- 09:00 이후 손절 발생 시, 포트폴리오 매니저는 손절 완료 즉시 슬랙에 **갱신된 현금 잔액**을 보고하며, 10:00 포지션 계산은 **가장 최근 슬랙 보고 기준 현금 잔액**을 사용한다

### 슬랙 현금 잔액 보고 누락 시 대응 _(킥오프 미팅 확정)_
- 누락 감지 시: 리스크 매니저가 API로 잔액 직접 조회 후 계산 진행
- 동시에: 슬랙에 "현금 잔액 보고 누락 — API 직접 조회로 대체" 알림 발송
- 포지션 계산이 불가한 경우(API 조회 실패 포함): 슬랙에 "포지션 계산 보류" 알림 → 팀장 에스컬레이션

## 6. 동시 돌파 시 우선순위 _(킥오프 미팅 확정)_

복수 종목이 swing_high를 동시에 돌파할 경우 아래 순서로 처리:

1. **리스크 비율이 낮은 종목 먼저** — `(swing_high - ma20) / swing_high` 값이 작은 순
2. 거래량 동반 여부 (강도가 명확한 종목)
3. 섹터 내 시가총액이 큰 종목 (유동성)

**정렬 주체 _(킥오프 미팅 확정)_**: 차트 분석가가 리스크 비율 포함 산출 → 리스크 매니저가 최종 정렬 확정 → 팀장 승인 → 포트폴리오 매니저 순서대로 집행

포트폴리오 슬롯(5개)이 다 차면 이후 종목은 당일 진입 포기.

## 7. 전체 운영 사이클

### 주간 사이클 개요

| 시간 | 이벤트 | 스크립트 | 빈도 |
|------|--------|---------|------|
| 월요일 07:30 | 팀 킥오프 미팅 | `team_kickoff.py` | 주 1회 |
| 평일 08:00 | 장시작 전 분석 | `orchestrator.py pre_open` | 매일 |
| 평일 09:00 | 익절 매도 | `orchestrator.py market_open` | 매일 |
| 평일 09:05 | 손절 모니터링 | `orchestrator.py stop_loss_monitor` | 매일 |
| 평일 10:00 | 고점 돌파 감시 → 매수 | `orchestrator.py entry_monitor` | 매일 |
| 주말 | 전략 토론 및 고도화 | `strategy_debate.py` | 주 1회 |

---

### 월요일 07:30 — 팀 킥오프 미팅 (`team_kickoff.py`)
- 에이전트 전원이 전체 매매 흐름을 함께 숙지
- 각자 역할·입출력·주의사항 재확인
- 팀장이 이번 주 운영 전 체크리스트 정리
- 결과: `data/kickoff_notes.md` 저장

> 전략이나 에이전트 `.md` 파일이 변경된 경우 해당 요일에 추가 실행 권장

### 평일 08:00 — 장시작 전 분석 (`pre_open`)
1. **섹터 선정**: 업종지수 1개월 수익률 기준 1개 선정, 신뢰도 판단
2. **후보 종목 선정**: 선정 섹터 내 3조건 필터링 → 투자 고려 대상
3. **보유 종목 점검**: 매수가/현재가/수익률/손절가 확인
4. **익절 대상 선정**: 현재가 < 20일선인 보유 종목
5. **슬랙 알림**: 선정 섹터, 투자 고려 대상, 익절 대상 보고

> **섹터 선정 보류 시**: 2~5단계 건너뜀, 보유 종목 관리만 진행

### 평일 09:00 — 익절 매도 (`market_open`)
1. **익절 대상 종목 시장가 매도** 실행
2. **슬랙 알림**: 매도 완료 + 현금 잔액 보고 (10:00 포지션 계산 기준)

### 평일 09:05~15:30 — 손절 모니터링 (`stop_loss_monitor`)
1. 보유 종목 현재가를 **60초 간격**으로 실시간 조회
2. 현재가 ≤ 손절가 도달 시 **즉시 시장가 매도** 실행
3. **슬랙 알림**: 손절 종목·수량·가격 + **갱신된 현금 잔액** 보고 (의무)
4. 보유 종목 없으면 즉시 종료
- `entry_monitor`(10:00~)와 병렬 실행, 역할은 완전히 분리

### 평일 10:00~10:30 — 고점 돌파 감시 → 매수 (`entry_monitor`)
1. 투자 고려 대상 종목 현재가 **실시간 감시**
2. `현재가 >= swing_high` 돌파 감지
3. **리스크 매니저**: 포지션 크기 계산 및 리스크 검토
4. **팀장**: 섹터·차트·리스크 의견 종합 → 매수 승인 또는 보류
5. 승인 시 **즉시 매수**, 손절가 = 매수 시점 20일선으로 설정
6. 10:30까지 진입 없으면 **"매수 없음"** 슬랙 보고

### 주말 — 전략 토론 및 고도화 (`strategy_debate.py`)
- 에이전트들이 정해진 주제로 토론 (`DEBATE_TOPICS` 편집 가능)
- 라운드 1: 각자 문제점·개선 의견 제시
- 라운드 2: 상호 검토 및 보완
- 팀장이 실행 가능한 개선안으로 종합
- 결과: `data/strategy_notes.md` 저장 → 필요 시 `.md` 파일에 반영

### 항시
- 보유 종목 현재가가 손절가 이하 도달 시 **즉시 매도** — 예외 없음
- 실행 주체: `stop_loss_monitor` (09:05~15:30, 60초 간격)
- 역할 분리: **리스크 매니저**(감지·트리거) → **포트폴리오 매니저**(시장가 매도 실행·슬랙 보고·현금 잔액 갱신). 매도 실패 시 팀장에게 즉시 에스컬레이션

## 8. 에이전트 구성 및 역할

| 에이전트 | 파일 | 판단 지침 | 주요 역할 |
|---------|------|----------|---------|
| 섹터 스카우트 | `agents/sector_scout.py` | `SECTOR_SCOUT.md` | 섹터 선정 (업종지수 수익률) |
| 차트 분석가 | `agents/chart_analyst.py` | `CHART_ANALYST.md` | 3조건 필터링, swing_high 산출, 실시간 감시 |
| 리스크 매니저 | `agents/risk_manager.py` | `RISK_MANAGER.md` | 포지션 크기 **계산·확정** (재계산 없음), 손절 감지·트리거 |
| 포트폴리오 매니저 | `agents/portfolio_manager.py` | `PORTFOLIO_MANAGER.md` | 매수/매도 주문 **실행**, 자금 배분 (포지션 재계산 안 함) |
| 팀장 | `orchestrator.py` (TeamLead) | `TEAM_LEAD.md` | 최종 매수 승인 또는 보류 |

### 팀장 매수 승인 기준
- 차트 분석가 3조건 모두 충족 ✓
- 리스크 매니저 "적정" 또는 "소량 진입 권장" ✓
- 섹터 강세 유지 중 ✓
- 포트폴리오 슬롯 여유 있음 ✓

→ 하나라도 미충족 시 **보류**
